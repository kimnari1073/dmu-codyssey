텍스트 파일과 이진 파일의 차이점 및 장단점

1. 텍스트 파일과 이진 파일 개요

📌 텍스트 파일 (Text File)

사람이 읽을 수 있는 형식으로 저장된 파일

일반적으로 .txt, .csv, .json 등의 확장자를 가짐

데이터가 문자(character)로 저장되며 줄바꿈(\n) 등의 가독성을 위한 기호가 포함됨

📌 이진 파일 (Binary File)

기계가 해석할 수 있는 0과 1의 데이터로 저장된 파일

.bin, .dat, .exe, .png, .mp3 등의 확장자를 가짐

문자뿐만 아니라 다양한 데이터(이미지, 오디오, 실행 파일 등)를 저장할 수 있음

2. 텍스트 파일과 이진 파일의 차이점

구분

텍스트 파일

이진 파일

저장 방식

사람이 읽을 수 있는 문자 형태

기계가 해석하는 0과 1의 바이너리 데이터

파일 크기

데이터 크기가 큼 (공백, 개행 포함)

크기가 작고 저장 공간을 효율적으로 사용

읽기/쓰기 속도

문자열 변환 과정이 있어 상대적으로 느림

변환 과정이 없어 빠름

수정 용이성

텍스트 편집기로 쉽게 수정 가능

직접 수정이 어려우며 전용 프로그램 필요

가독성

사람이 직접 읽고 이해 가능

사람이 읽기 어려움

호환성

다양한 프로그램에서 쉽게 열 수 있음

특정 프로그램에서만 해석 가능

3. 텍스트 파일과 이진 파일의 장단점

✅ 텍스트 파일의 장단점

장점

사람이 읽고 편집하기 쉬움

다양한 프로그램에서 열 수 있음 (메모장, VS Code, 터미널 등)

버전 관리(Git)에서 차이를 쉽게 비교 가능

단점

데이터 크기가 큼 (불필요한 공백 및 개행 포함)

읽기/쓰기가 상대적으로 느림

정형화되지 않은 데이터를 저장하기 어려움

✅ 이진 파일의 장단점

장점

저장 공간을 효율적으로 사용 (불필요한 공백 제거)

읽기/쓰기 속도가 빠름

텍스트뿐만 아니라 이미지, 영상, 오디오 등 다양한 데이터 저장 가능

단점

사람이 직접 읽거나 수정하기 어려움

특정 프로그램에서만 열고 수정 가능

버전 관리에서 차이점을 확인하기 어려움
